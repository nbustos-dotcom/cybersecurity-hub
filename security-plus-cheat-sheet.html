Domain 1 — 12%

General Security Concepts

  • CIA Triad Confidentiality, Integrity, Availability
  • AAA Authentication, Authorization, Accounting
  • Zero Trust Never trust, always verify
  • Defense in Depth Layered security controls
  • Least Privilege Minimum access needed
  • Separation of Duties No single point of control
  • Security Control Types Technical, Admin, Physical
  • Control Functions Preventive, Detective, Corrective, Deterrent
Domain 2 — 22%

Threats, Vulnerabilities & Mitigations

  • Malware types Virus, Worm, Trojan, Ransomware, Rootkit, Spyware
  • Social Engineering Phishing, Vishing, Smishing, Pretexting, Baiting
  • Password Attacks Brute force, Dictionary, Spraying, Credential stuffing
  • MitM / On-Path Intercepts communication between parties
  • SQL Injection Malicious SQL via input fields
  • XSS Inject scripts into trusted sites
  • CSRF Force authenticated user actions
  • Buffer Overflow Overwrite memory to execute code
  • Zero-Day Exploit before patch exists
  • Insider Threat Malicious or negligent internal user
Domain 3 — 18%

Security Architecture

  • DMZ Buffer zone between public and internal network
  • VLAN Logical network segmentation
  • NAT Maps private IPs to public IPs
  • Firewall types Packet filter, Stateful, NGFW, Proxy
  • IDS vs IPS IDS detects only; IPS detects + blocks
  • Proxy Server Intermediary between client and internet
  • Load Balancer Distributes traffic across servers
  • Honeypot Decoy system to attract attackers
  • Air Gap Physically isolated network
  • SD-WAN Software-defined wide area network
Domain 4 — 28%

Security Operations

  • Incident Response Prepare → Identify → Contain → Eradicate → Recover → Lessons
  • SIEM Aggregates and correlates security logs
  • EDR Endpoint detection and response
  • Vulnerability Scan Non-intrusive; identifies known weaknesses
  • Penetration Test Authorized simulated attack
  • Digital Forensics Order of volatility; chain of custody
  • Log Management Collection, retention, analysis
  • Change Management Documented, approved system changes
  • Identity & Access MFA, SSO, PAM, RBAC, ABAC
  • Data Loss Prevention DLP monitors and blocks data exfil
Domain 5 — 20%

Security Program Management

  • Risk Terminology Threat × Vulnerability × Impact = Risk
  • Risk Treatments Accept, Transfer, Avoid, Mitigate
  • BCP / DRP Business continuity and disaster recovery
  • RTO / RPO Recovery time and point objectives
  • Data Classifications Public, Internal, Confidential, Restricted
  • Regulations GDPR, HIPAA, PCI-DSS, SOX, FERPA
  • Security Awareness Training, phishing simulations, policies
  • Vendor Management Third-party risk, supply chain security

Quick Reference

Cryptography Essentials

  • Symmetric Same key to encrypt/decrypt. AES, 3DES. Fast.
  • Asymmetric Public/private key pair. RSA, ECC. Slower.
  • Hashing One-way. MD5 (broken), SHA-256, SHA-3
  • Digital Signature Sign with private key, verify with public
  • PKI / CA Certificate authorities issue/validate certs
  • TLS Handshake Negotiate cipher → exchange keys → encrypt
  • Key Exchange Diffie-Hellman, ECDH
  • Salting Random data added before hashing passwords
